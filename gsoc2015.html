RiVal GSoC 2015 Ideas

This page contains ideas of RiVal contributions in the context of GSoC 2015. These ideas should not be seen as the only possible GSoC projects, if you want to expand on one of the provided ideas, or propose something else, do get in touch with us at rival@recommenders.net

Below is an unordered list of possible GSoC RiVal projects:

  • Web Interface/RESTful API
  • Statistical Significance+
  • Non-traditional evaluation metrics
  • Stream-based evaluation
  • Extending support of recommendation frameworks

 

Project: Web Interface/RESTful API

Brief explanation: RiVal currently runs as a CLI program or is used as a Maven module. This project will set RiVal up as a Web service allowing for testing and evaluation to be performed over a RESTful API.

Expected Results: A RiVal webapp which will allow for RESTful interactions with RiVal.

Knowledge Prerequisite: Java and web programming.

Skill: Medium

Mentor: Alan Said or Alejandro Bellogín

 

 

Project: Statistical Significance+

Brief explanation: RiVal currently does not calculate statistical significance for the available metrics. This project will set up a statistics module allowing for various statistical tests (e.g. student's T-Test, etc.) to be performed on the generated values.

Expected Results: A RiVal statistics module containing (at least) a set of basic stastistical measures to be calculated based on the data and resuts.

Knowledge Prerequisite: Java, statistics, machine learning

Skill: Medium to high

Mentor: Alejandro Bellogín or Alan Said

 

 

Project: Non-traditional evaluation metrics

Brief explanation: In order to evaluate subjective experience of recommendation algorithms, metrics such as serendipity and novelty are used. This project will create metrics based on current state-of-the-art research literature for evaluating recommender systems with these metrics.

Expected Results: A RiVal module, or expansion of the current evaluaiton module to support non-traditional evaluation metrics.

Knowledge Prerequisite: Java, recommender systems

Skill: Medium to high

Mentor: Alan Said or Alejandro Bellogín

 

 

Project: Stream-based evaluation

Brief explanation: RiVal currently reads data from files on disk, this project will create an interface for RiVal to receive recommendations live - as they are created - and present results for certain timespans.

Expected results: an evaluation architecture and module which allows for live, continuous, and burst-based evaluation of incoming recommendations.

Knowledge Prerequisite: Java

Skill: Medium

Mentor: Alejandro Bellogín or Alan Said

 

 

Project: Extending support of recommendation frameworks

Brief explanation: RiVal currently supports three recommendation frameworks: Apache Mahout, MyMediaLite, and LensKit; this project will adapt the interfaces available in RiVal to support other recommendation libraries such as GraphChi/Lab, neo4j, etc.

Expected results: a module adding support for a recommendation library from one of the open source projects that is not already available in RiVal.

Knowledge Prerequisite: Java

Skill: Medium

Mentor: Alejandro Bellogín or Alan Said
